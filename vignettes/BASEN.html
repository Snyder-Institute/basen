BASEN

Heewon Seo

March 11, 2026

Abstract

Comparing taxonomic abundance across long-read metagenomic samples is challenging because abundance summaries can be influenced by sequencing depth, read-length variation, taxonomic assignment characteristics, and reference genome size. In Oxford Nanopore data, broad read-length distributions further complicate the interpretation of count-based summaries. BASEN is a framework for species-level abundance estimation from Kraken2 outputs that aggregates species-assigned k-mer evidence, normalizes this signal by effective reference genome length, and generates genome-size-aware relative abundance estimates. BASEN also provides read-level compositional quality-control metrics, including assigned and unassigned k-mer proportions and Shannon diversity of k-mer assignments, to support identification of low-information reads. Together, these features improve the robustness and interpretability of species-level abundance estimates in long-read metagenomic analyses.

Overview

BASEN (Base-level Abundance estimation with Species-assigned Evidence using Nanopore) computes genome-size-normalized, species-level relative abundance from Kraken2 classification outputs. BASEN aggregates species-assigned k-mer evidence across classified reads, normalizes the resulting signal by effective reference genome length, and scales the normalized values within each sample to produce relative abundance estimates. In addition, BASEN provides read-level k-mer composition metrics that support compositional quality control prior to abundance estimation. This vignette demonstrates a typical BASEN workflow using example files shipped with the package under inst/testdata.

Workflow

0. Preparation

A. Construct a custom database

  • To ensure correct taxonomic assignment, reference genome FASTA headers should include a Kraken2-recognizable TaxID in the following format:
|kraken:taxid|XXXXXX
  • An example workflow for building a custom Kraken2 database is shown below.
# Do not run
k2 download-taxonomy --db MYCUSTOM
k2 add-to-library --file XXXXXX.fasta --db MYCUSTOM
k2 build --db MYCUSTOM

B. Run Kraken2 classification

  • Run Kraken2 using the custom database and generate both .report and .kraken output files for each sample.
# Do not run
export DB_DIR=/path/to/MYCUSTOM
export PROJECT=/path/to/PROJECT
export FASTQ_DIR=/path/to/FASTQ

k2 classify \
  --db $DB_DIR \
  --report $PROJECT/REPORT/SAMPLE_x.report \
  --output $PROJECT/KRAKEN/SAMPLE_x.kraken \
  $FASTQ_DIR/SAMPLE_x.fastq.gz

C. Generate genome size statistics

  • BASEN requires genome size information for abundance normalization. The auxiliary script get_genome_length_from_fasta.sh, located under inst/scripts, can be used to extract genome lengths directly from FASTA files.
  • The script outputs a tab-delimited file without a header, containing the following columns:
    1. TaxID
    2. Total genome length in base pairs
    3. Accession
    4. Description
  • Only the first two columns, TaxID and total genome length, are required by BASEN.
sh get_genome_length_from_fasta.sh *.fa > genome_size.txt

1. Read-level compositional quality control

Long-read taxonomic assignments can be ambiguous. Before quantifying sample-level abundance, BASEN can profile the k-mer composition of individual reads to identify low-information classifications and define empirical filtering thresholds.

A. Load the package and example paths

  • First, load the packages and define the absolute paths to your Kraken classification files and genome size statistics.
library(basen)
library(data.table)
library(ggplot2)

kraken_dir <- system.file("testdata", package = "basen")
genome_stats_file <- system.file(
  "testdata", "example_genome_size.txt",
  package = "basen"
)

B. Profile k-mer composition for a single sample

  • The profile_kmer_composition() function parses a Kraken2 *.kraken file and computes per-read metrics, including total k-mers, assigned k-mers, unassigned k-mers, assigned and unassigned proportions, and Shannon diversity of the k-mer assignment profile.
kraken_file <- file.path(kraken_dir, "exampleA.kraken")

qc_metrics <- profile_kmer_composition(
  kraken_file = kraken_file,
  k_mer_length = 35
)

    kraken_file = file,
    k_mer_length = 35

  metrics[, sample_id := sample_name]
  metrics

2. Species-level abundance estimation

After defining a read-level QC filter, BASEN can estimate genome-size-aware species-level relative abundance using the retained reads.

A. Calculate relative abundance

  • The kraken_relative_abundance() function aggregates species-assigned k-mer evidence across classified reads, normalizes this signal by effective reference genome length, and scales the resulting coverage proxies within each sample to generate relative abundance estimates.
  • If a read filter is provided, only retained reads contribute to abundance estimation.
  • Interpretation of the output columns:
    1. bases_assigned: total number of species-assigned k-mers
    2. reads_assigned: number of reads assigned to the species
    3. genome_size_bp: effective reference genome size used for normalization
    4. coverage_proxy: genome-length-normalized sequencing signal
    5. relative_abundance: within-sample normalized species-level abundance
ra_dt <- kraken_relative_abundance(
  report_dir = kraken_dir,
  kraken_dir = kraken_dir,
  genome_stats_file = genome_stats_file,
  read_filter = hq_reads_list,
  verbose = TRUE
)

3. Cross-sample normalization

BASEN also provides an optional utility for estimating sample-specific size factors from coverage-proxy matrices to support downstream cross-sample normalization. This step is not part of the core BASEN abundance-estimation procedure, but may be useful for comparative analyses across samples with different sequencing depths.

A. Calculate size factors

  • The calc_size_factors() function implements a median-ratio normalization strategy. It constructs a reference across taxa, computes per-sample ratios relative to that reference, and summarizes those ratios into sample-specific size factors. This approach is more robust to highly abundant taxa than simple total-sum scaling.
  • Importantly, size factors should be calculated from coverage_proxy values rather than relative_abundance, because relative abundance is already compositional and does not retain the total-depth signal required for this normalization step.
abundance_mat <- dcast(
  ra_dt,
  taxid ~ sample_id,
  value.var = "coverage_proxy",
  fill = 0
)

coverage_mat <- as.matrix(abundance_mat[, -1, with = FALSE])
rownames(coverage_mat) <- abundance_mat$taxid

sf_median <- calc_size_factors(
  x = coverage_mat,
  method = "median"
)
#> Execution statistics for calc_size_factors:
#> Total matrices processed: 1
#> Taxa used for reference: 6
#> Size factors calculated: 2

print(sf_median)
#>  exampleA  exampleB 
#> 1.0138142 0.9928107

B. Apply normalization

  • Once the size factors are calculated, a normalized coverage-proxy matrix can be obtained by dividing each sample by its corresponding size factor.
normalized_mat <- sweep(coverage_mat, 2, sf_median, "/")
